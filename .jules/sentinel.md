## 2024-06-11 - Missing Authentication on Settings Update API
**Vulnerability:** The `/api/settings/update` FastAPI endpoint was completely unauthenticated. Any user (or attacker) could send a POST request with arbitrary `guild_id` and channel IDs to overwrite server settings, or even lock down channels in servers where the bot is present.
**Learning:** The initial implementation relied solely on the frontend logic to "hide" the dashboard until the user logged in via OAuth2, ignoring that APIs can be called directly via tools like curl or Postman without going through the UI.
**Prevention:** Always validate authorization directly at the API endpoint level for any state-mutating actions (e.g., database updates or executing bot actions). Extract the user's `access_token` from the `Authorization` header and dynamically verify their identity and permissions against the Discord API (e.g., checking `0x8` Administrator flag in the specific `guild_id`) before processing the request.
