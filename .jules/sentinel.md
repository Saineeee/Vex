## 2024-05-18 - Missing Authentication and Authorization on Settings Update
**Vulnerability:** The `/api/settings/update` FastAPI endpoint accepts `SettingsUpdate` payloads and modifies database configurations or performs actions (like channel lockdown) for any `guild_id` without verifying the caller's identity or permissions. An attacker could exploit this to alter settings or lock down channels in any server the bot is in.
**Learning:** FastAPI routes are public by default. The OAuth2 login endpoint (`verify_login`) validated the user but did not return the `access_token` for the frontend to include in subsequent requests, leading developers to leave the update endpoint unprotected.
**Prevention:** Always require and validate an authorization token (e.g., Bearer token) on endpoints that modify state. Furthermore, enforce strict authorization checks (e.g., ensuring the user has Administrator permissions in the target `guild_id`) before performing any action on behalf of a user.
