## 2024-04-28 - [CRITICAL] Missing Authorization on FastAPI Dashboard Endpoint
**Vulnerability:** The `/api/settings/update` endpoint allowed unauthenticated, unverified POST requests to modify critical server configurations and trigger immediate actions (like locking down a Discord channel) simply by knowing or guessing a `guild_id`.
**Learning:** The FastAPI route lacked dependency injection or manual checking for a Discord access token. The login route worked properly to retrieve tokens, but the frontend didn't store or pass it to protected routes, creating a severe IDOR (Insecure Direct Object Reference) and missing authentication vulnerability.
**Prevention:** Always require and validate Bearer tokens for API endpoints that modify state. Verify the token with the identity provider (Discord API `/users/@me/guilds`) and ensure the identity possesses the necessary permissions (e.g., Administrator `0x8`) for the specific object (`guild_id`) being modified before processing the request.
